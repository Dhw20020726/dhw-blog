Docusaurus 知识点总结

本文汇总了搭建本项目过程中涉及的所有核心概念，按讨论顺序整理。

1. 项目结构​

dhw-blog/
├── docs/                    # Markdown 文章，由 docs 插件渲染
│   └── angular/
│       └── intro.md         # 自动生成路由 /docs/angular/intro
├── src/
│   ├── pages/               # 文件即路由，一个 .jsx → 一个 URL
│   │   ├── index.jsx        # 对应 /
│   │   └── docs/
│   │       └── index.jsx    # 对应 /docs
│   ├── theme/               # Swizzle 覆盖默认主题组件
│   │   ├── Root.jsx         # 全局最外层包裹
│   │   └── MDXComponents.jsx # Markdown → React 组件映射
│   └── css/
│       └── custom.css       # 全局自定义样式
├── plugins/                 # 自定义 remark 插件
├── scripts/                 # 构建脚本（Algolia 索引等）
├── static/                  # 静态资源（图片、图标）
├── docusaurus.config.js     # 主配置（站点名、baseUrl、导航栏等）
├── sidebars.js              # 侧边栏配置
└── .github/workflows/       # CI/CD 自动部署

2. 路由​

两种路由方式​

目录路由方式例子
src/pages/文件即路由src/pages/index.jsx → /
docs/插件驱动docs/angular/intro.md → /docs/angular/intro
  • src/pages/ 下文件名 = URL 路径。index.jsx 等于目录的默认页（类似 index.html）
  • docs/ 不需要手动配置路由，Docusaurus docs 插件自动把 .md 文件渲染到 /docs/ 路径下
  • 访问 /docs 时，默认跳转行为由 src/pages/docs/index.jsx 控制

为什么用户访问 /docs 会触发 src/pages/docs/index.jsx？​

Docusaurus 基于文件系统自动生成路由：src/pages/ 下的文件结构直接映射为 URL。index.jsx 等于目录首页，和 index.html 同理。

3. docusaurus.config.js 核心配置​

module.exports = {
  title: '站点标题',               // 浏览器标签栏文字
  tagline: '副标题',               // 首页描述
  url: 'https://xxx.github.io',   // 部署域名
  baseUrl: '/dhw-blog/',          // 子路径，如果是项目站点必须加仓库名
  organizationName: 'GitHub用户名',
  projectName: '仓库名',

  presets: [
    ['classic', {
      docs: {
        sidebarPath: require.resolve('./sidebars.js'),   // 侧边栏配置文件
        routeBasePath: '/docs',                           // docs 文档挂载的 URL 路径
        remarkPlugins: [],                                // Markdown 处理插件
      },
    }],
  ],

  themeConfig: {
    algolia: {                        // Algolia 搜索
      appId: 'xxx',
      apiKey: 'xxx',                  // Search API Key（可公开）
      indexName: 'xxx',
    },
    navbar: {
      title: '导航栏标题',
      hideOnScroll: true,             // 向下滚动隐藏导航栏
    },
  },
};

baseUrl 特别注意​

baseUrl 告诉 Docusaurus "网站部署在域名的哪个子路径下"。

最终 URL 是怎么拼出来的：

url                 +  baseUrl         +  页面路径     =  完整 URL
https://xxx.github.io  /dhw-blog/         docs/intro   →  https://xxx.github.io/dhw-blog/docs/intro

这三个部分各管各的：

  • url：域名（协议 + 主机名），不带尾斜杠
  • baseUrl：域名后面要补的前缀路径，必须首尾都有 /（比如 /dhw-blog/，不是 dhw-blog）
  • 页面路径：Docusaurus 根据你的 docs/ 和 src/pages/ 自动生成

为什么会有 baseUrl 这个东西？

因为有些网站不是放在域名根目录的。比如 GitHub Pages：

https://dhw20020726.github.io/           ← 这是用户站点，域名的根
https://dhw20020726.github.io/dhw-blog/  ← 这是项目站点，多了一层 /dhw-blog/

GitHub 给每个仓库分配的地址格式是 用户名.github.io/仓库名/，所以你的网页不在根目录，而在 /dhw-blog/ 这个子目录下。

两种情况：

场景例子baseUrl原因
用户站点dhw20020726.github.io'/'网站就在域名根目录
项目站点dhw20020726.github.io/dhw-blog'/dhw-blog/'网站前面多了一层仓库名

baseUrl 影响什么？

Docusaurus 内部所有路径都基于 baseUrl 生成：

  • CSS / JS 文件引用：/dhw-blog/assets/css/xxx.css
  • 页面间链接：/dhw-blog/docs/angular/intro
  • 图片路径：/dhw-blog/img/logo.svg
  • 搜索索引、favicon 等

设错会怎样？

baseUrl: '/'（但仓库叫 dhw-blog）→ 部署后所有静态资源 404，页面空白，控制台报错 "Docusaurus site did not load properly"。因为浏览器去 https://xxx.github.io/assets/css/xxx.css 找文件，但实际文件在 https://xxx.github.io/dhw-blog/assets/css/xxx.css。

怎么验证？

你仓库名如果是 dhw-blog，部署地址就是 https://用户名.github.io/dhw-blog/，那 baseUrl 就必须是 '/dhw-blog/'。

4. sidebars.js​

module.exports = {
  docsSidebar: [     // 名字随意，Docusaurus 取都出的第一个 key
    {
      type: 'category',            // 可折叠的目录分类
      label: '分类名',             // 侧边栏显示文字
      collapsed: false,            // false = 默认展开，true = 收起
      link: {
        type: 'generated-index',   // 自动生成分类首页
        title: '分类首页标题',
        description: '描述',
      },
      items: [
        '子目录/文件名',             // 简写，等价于 { type: 'doc', id: '子目录/文件名' }
      ],
    },
  ],
};

侧边栏四种 type​

type含义用途
category可折叠的文件夹套娃分类，子项可以是 doc、link
doc单篇文章items 里的字符串简写就是这个
link外部链接跳转外部网站，不渲染 Markdown
autogenerated自动扫描目录省事，不用手动列所有文件
  • items 里的 'angular/intro' 会自动映射到 docs/angular/intro.md
  • 路径不需要写 .md 后缀，相对于 docs/ 目录

category 的 link 属性​

link 控制点击分类名时的行为：

有 link 时：

{
  type: 'category',
  label: 'React',
  link: { type: 'generated-index', title: 'React', description: '...' },
  items: ['react/intro', 'react/hooks'],
}
  • Docusaurus 自动生成分类首页 /docs/category/react，列出该分类下所有文章
  • 点击分类名 → 跳转到这个分类首页
  • 首页卡片和重定向可以指向 /docs/category/xxx

没有 link 时：

{
  type: 'category',
  label: 'React',
  items: ['react/intro', 'react/hooks'],
}
  • 分类名只是一个折叠/展开按钮，没有自己的页面
  • /docs/category/xxx 不存在，指向它会 404
  • 首页卡片只能链接到具体文章，比如 /docs/react/intro
  • 侧边栏仍然正常显示分类和子项，只是分类名不可点击跳转

@type JSDoc 注释​

/** @type {import('@docusaurus/plugin-content-docs').SidebarsConfig} */
const sidebars = {
  docsSidebar: [...],
};
  • 这是 JSDoc 类型注释，不是代码，不影响运行
  • 告诉编辑器 "这个变量是 SidebarsConfig 类型"
  • 效果：在 VS Code 里写 items、type 时有自动补全，写错了有红色波浪线
  • import('...') 是 TypeScript 语法，JSDoc 借用它来动态引入类型定义（@docusaurus/plugin-content-docs 包里自带了 .d.ts 类型文件）
  • 同理 docusaurus.config.js 第一行的 // @ts-check + /** @type {import('@docusaurus/types').Config} */ 也是同一套用法 — 纯 JS 文件获得 TypeScript 的类型检查能力，无需编译

5. Theme Swizzling（主题覆盖）​

src/theme/ 下的文件名如果和 Docusaurus 默认主题组件同名，会自动替换默认的。

页面加载顺序（类似 Angular 的 index.html → main.ts → AppComponent）​

Docusaurus/React 也有自己的加载链条：

浏览器请求 URL
  ↓
1. 静态 HTML（预渲染的壳子，含 <div id="__docusaurus">）
  ↓
2. 客户端 JS bundle 下载并执行（@docusaurus/core 入口）
  ↓
3. React 接管页面，开始构建组件树，从外向内渲染：
  ↓
<Root>                          ← 你的 src/theme/Root.jsx（覆盖默认）
  └── <Layout>                  ← Docusaurus 默认（导航栏 + 侧边栏 + 内容壳）
        └── <你的页面组件>       ← src/pages/index.jsx 或 docs 插件渲染的 Markdown
  ↓
4. 每个组件渲染完成后，内部的 useEffect 依次触发

对比 Angular：

步骤AngularDocusaurus / React
HTML 入口index.html构建生成的静态 HTML（含 <div id="__docusaurus">）
启动模块main.ts → bootstrapModule(AppModule)@docusaurus/core 客户端入口 → ReactDOM.render()
根组件AppComponentRoot（可 swizzle 覆盖）→ Layout
路由解析RouterModule 匹配 URL → 加载对应 ComponentReact Router 匹配 URL → 渲染对应页面
页面内容对应 Component 的模板src/pages/ 下的 React 组件，或 docs 插件渲染 Markdown
生命周期ngOnInituseEffect(() => {}, [])

Root.jsx — 全局根组件​

import React from 'react';                    // 写 JSX 必须引入
export default function Root({ children }) {  // children = 所有页面内容
  return children;                            // 直接返回，什么都不改
}
  • 非必须：src/theme/Root.jsx 是 swizzle 覆盖，删掉后 Docusaurus 用自己的默认 Root，一切正常运行
  • 它是 React 组件树的最外层，所有页面的公共祖先
  • 常见用途：全局埋点（page view tracking）、主题 provider、CSS 变量注入、上次阅读位置恢复等
  • 当前项目什么都没做，{children} 原样透传 — 纯为以后扩展留入口
  • { children } 是 React 的 children prop，代表组件标签内部的所有内容

MDXComponents.jsx — 自定义 Markdown 渲染​

Docusaurus 渲染 Markdown 时，每个元素对应一个 React 组件：

Markdown                      →  React 组件
## 标题                        →  <h2>标题</h2>
| 表格 | 内容 |                →  <table>...</table>
`代码`                         →  <code>代码</code>

MDXComponents 就是一张 映射表，你可以把默认的组件替换成自定义的：

import MDXComponents from '@theme-original/MDXComponents';
export default { ...MDXComponents, h2: MyH2 };
//                                  ↑    ↑
//                              元素名  你自定义的组件

具体例子：给所有 <h2> 加一个可点击的锚点链接

默认 h2 渲染效果：    ## 安装步骤

替换后 h2 渲染效果：  ## 安装步骤 #
                              ↑ 鼠标悬停出现，点它可以复制当前标题的链接
import MDXComponents from '@theme-original/MDXComponents';
import React from 'react';

function MyH2(props) {
  const id = props.id;  // Docusaurus 自动生成的锚点 id
  return (
    <h2 {...props}>                {/* 保留默认 h2 的所有属性 */}
      {props.children}             {/* 原标题文字 */}
      <a href={`#${id}`} className="anchor-link">#</a>  {/* 追加锚点链接 */}
    </h2>
  );
}

export default { ...MDXComponents, h2: MyH2 };

逐行解释：

  • { ...MDXComponents, h2: MyH2 }：把默认映射表展开，只把其中 h2 替换成 MyH2，其他元素（h1、table、code 等）保持默认
  • function MyH2(props)：拿到 Docusaurus 传给 h2 的属性（id、className、children 等）
  • {...props}：把原属性全部透传给 <h2>，不丢任何东西
  • {props.children}：渲染原标题文字
  • <a href=...>#</a>：追加自己的内容

可以替换的元素列表：h1 h2 h3 h4 h5 h6 a img table code pre p ul ol li blockquote hr em strong del input 等。

6. React 基础知识点​

useEffect — 渲染后执行副作用​

useEffect(fn, [依赖]);
  • 第一个参数：渲染完成后执行的函数（副作用 = 操作 DOM、跳转、发请求等 UI 渲染之外的事）
  • 第二个参数（依赖数组）：只有数组中变量变化才重新执行。[] = 永远只执行一次（首次渲染后）
  • 不用 useEffect 直接在组件函数体里写跳转会导致每次重渲染都跳，无限循环

window.location.replace() vs window.location.href​

// 普通赋值 — 追加历史记录
window.location.href = '/new-url';
// 历史： [首页] → [/old] → [/new-url]
// 后退： /old

// replace — 替换当前记录
window.location.replace('/new-url');
// 历史： [首页] → [/new-url]
// 后退： 首页

replace 把当前页面从历史栈里删掉，用户按后退不会回到中转页。适合跳转页、登录后重定向等场景。

箭头函数​

() => { ... }    // 等价于 function() { ... }
(x) => x * 2     // 等价于 function(x) { return x * 2; }

解构赋值​

import { useEffect } from 'react';
// 只要 react 包里名叫 useEffect 的那个，其他不要

JSX 中的 style​

style={{ padding: '4rem 1.5rem', textAlign: 'center' }}
//    ↑ 这层 {} = 告诉 JSX "里面是 JS 表达式"
//     ↑ 这层 {} = 这是一个 JS 对象

7. Algolia 搜索​

三部分配合​

  1. 配置（docusaurus.config.js）：themeConfig.algolia 填入 appId、apiKey、indexName
  2. 索引上传（scripts/algoliaIndex.js）：扫描 docs/ → 解析 Markdown → 按标题切块 → 上传到 Algolia
  3. 前端弹窗（@docsearch/react）：用户搜索 → 浏览器直连 Algolia API → 返回结果

CI 环境变量​

构建时 docusaurus.config.js 需要读到 ALGOLIA_SEARCH_API_KEY 才能生成搜索栏，部署 workflow 必须在 build 步骤 注入环境变量：

- name: Build website
  env:
    ALGOLIA_APP_ID: ${{ secrets.xxx }}
    ALGOLIA_SEARCH_API_KEY: ${{ secrets.xxx }}
    ALGOLIA_INDEX_NAME: ${{ secrets.xxx }}
  run: npm run build
  • ALGOLIA_SEARCH_API_KEY 是前端只读 key，可公开，嵌入到 HTML 里
  • ALGOLIA_ADMIN_API_KEY 是后端写 key，只能出现在 CI secrets，绝对不能暴露到前端

replaceAllObjects 的 safe: true​

会在 Algolia 创建临时索引 xxx_tmp_xxxx，上传完再替换原索引。如果之前的临时索引没清理，后续上传会变慢。第一次上传或用 clearObjects + saveObjects 替代也可以。

8. CI / CD（GitHub Actions）​

name: Deploy to GitHub Pages

on:
  push:
    branches: [main]              # main 分支 push 时触发

jobs:
  deploy:
    steps:
      - uses: actions/checkout@v4        # 拉代码
      - uses: actions/setup-node@v4       # 装 Node.js
      - run: npm ci                       # 装依赖
      - run: npm run build                # 构建（注入了 Algolia env）
      - run: npm run index:all            # 上传搜索索引
      - uses: peaceiris/actions-gh-pages@v4  # 部署到 gh-pages 分支

GitHub Pages 设置中 Source 选 gh-pages 分支、/ (root) 目录。

10. 添加新文章的步骤​

  1. 在 docs/angular/（或 springboot/、react/）下新建 .md 文件
  2. 写上 frontmatter： 
    ---
    title: 文章标题
    ---
    正文...
  3. 在 sidebars.js 对应分类的 items 里加上文件路径（不带 .md）
  4. git commit && git push，CI 自动部署
